# Phase A — Operationalization

**Status:** ⬜ Not started  
**Horizon:** v1.1.0  
**Hard blockers:** Phase 12 (v1.0.0 tag), ADR-0006 (evidence storage), ADR-0007 (waiver governance)

## Goal

Transform the compliance system from a well-designed data model into a running system that produces real evidence from real CI runs. After Phase A, every push to `platform-compliance` automatically generates evidence records, an assessment report, and a gate evaluation — without human intervention.

## Why this comes before tooling (Phase B)

The `plt` CLI and developer experience depend on having real evidence and real assessment reports to work with. Building the CLI before the pipeline runs is building the dashboard before the data exists.

## Acceptance criteria
- A PR to `platform-compliance` on GitHub triggers the compliance workflow
- The workflow completes all 7 jobs without placeholder substitution
- Evidence records exist in the ledger for the assessed commit
- Assessment report is generated by CI, not manually authored
- Gate evaluation correctly blocks a deliberately failing PR (test: remove README, verify DOC-001 fails)
- `v1.1.0` can be tagged with a passing self-assessment from CI
